# 회비 관리 기능 법적 규제 정리

## 개요

동아리 회비를 다루는 방식에 따라 법적 규제가 달라집니다. 이 문서는 우학동에서 회비 관련 기능을 구현할 때 고려해야 할 법적 사항을 정리합니다.

---

## 규제가 필요한 구조 (피해야 함)

### 정산 대행 구조

```
동아리원 → 우학동(결제) → 동아리(정산)
```

이 구조는 **전자금융업** 등록이 필요합니다.

| 라이선스 | 최소 자본금 | 소요 기간 |
|----------|-------------|-----------|
| 전자지급결제대행업 (PG) | 30억원 | 6개월~1년 |
| 결제대금예치업 | 20억원 | 6개월~1년 |

### 충전금 보유 구조

```
동아리원이 우학동에 돈 충전 → 우학동이 보유 → 필요시 사용
```

| 라이선스 | 최소 자본금 | 비고 |
|----------|-------------|------|
| 선불전자지급수단 발행업 | 20억원 | 스타벅스, 카카오페이 모델 |

### 금융 데이터 자동 연동

```
동아리 카드/계좌 → 은행 API → 우학동 → 동아리원 열람
```

| 라이선스 | 최소 자본금 | 비고 |
|----------|-------------|------|
| 마이데이터 사업자 | 5억원 | ISMS 인증 필수 |

---

## 규제 없이 가능한 구조

### 1. 회비 수기 입력

```
동아리 임원이 회비 납부 현황을 직접 입력
```

- 단순 장부/가계부 기능
- 금융 데이터 연동 없음
- **전자금융업 등록 불필요**

### 2. 송금 링크 생성

```
우학동이 카카오페이/토스 송금 링크 생성 → 동아리원이 링크로 직접 송금
```

- 돈이 우학동을 거치지 않음
- 임원 계좌로 직접 입금
- **전자금융업 등록 불필요**

### 3. 명세서 파일 업로드

```
임원이 카드사/은행에서 CSV/PDF 다운로드 → 우학동에 업로드
```

- 임원이 직접 자신의 데이터를 업로드
- API 자동 연동 아님
- **마이데이터 등록 불필요**

## 우학동 구현 방향

### Pro 플랜 회비 관리 기능

| 기능 | 구현 방식 | 법적 이슈 |
|------|-----------|-----------|
| 회비 납부 현황 | 수기 입력 | 없음 |
| 회비 수납 | 송금 링크 생성 (카카오페이/토스) | 없음 |
| 수입/지출 내역 | 수기 입력 또는 파일 업로드 | 없음 |
| 회계 보고서 | 입력된 데이터 기반 자동 생성 | 없음 |

### 주의사항

1. **"정산"이라는 용어 사용 금지**: 정산은 우학동이 돈을 받아서 나눠준다는 의미로 오해될 수 있음
2. **돈의 흐름에 우학동이 포함되면 안 됨**: 항상 동아리원 → 동아리 직접 송금 구조
3. **은행/카드사 API 직접 연동 금지**: 마이데이터 라이선스 필요

## 향후 확장 시 고려사항

### 자동 정산 기능이 필요한 경우

1. **투자 유치**: 시리즈 A 50억원+ 규모
2. **전자금융업 등록**: 자본금 30억원 + 심사 6개월~1년
3. **보안 인증**: ISMS 인증 (3-6개월, 수천만원)

### 대안: 규제 샌드박스

- 금융위원회 혁신금융서비스 지정 신청
- 2년간 규제 면제 테스트 가능
- 심사 통과 필요

### 대안: 기존 서비스 연동

- 토스 동아리통장 연동/안내
- 카카오페이 송금 기능 활용
